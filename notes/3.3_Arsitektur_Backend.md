# 3.3 Arsitektur Backend (Server-Side)

Bagian peladen (backend) dari aplikasi DAFA dibangun menggunakan lingkungan *runtime* **Node.js** dan *framework* **Express.js**. Pemilihan Express.js didasari oleh kemampuannya memproses permintaan HTTP dengan cepat dan efisien melalui sistem lintasan (*routing*) dan fungsi penengah (*middleware*). 

## 3.3.1 Struktur Pengamanan dan Autentikasi

Arsitektur backend mengimplementasikan dua lapis pengamanan krusial:
1. **Kriptografi Sandi (Password Hashing)**
   Library `bcryptjs` digunakan pada lapisan pendaftaran atau pembaruan akun pengguna. Password *plain text* akan diubah menjadi untaian karakter acak dengan teknik *salting*. Hal ini memastikan bahwa data sandi tidak dapat dibaca meskipun basis data berhasil diretas.
2. **JSON Web Token (JWT) Authorization**
   Backend tidak mengandalkan otentikasi *stateful* (berbasis memori server). Setelah kredensial login tervalidasi, server menerbitkan (*sign*) token `jsonwebtoken` yang mengenkapsulasi *payload* data krusial seperti identitas pengguna dan peran (Admin/Client). Token ini ditanamkan (*embedded*) di dalam *HTTP-only cookies* agar terhindar dari serangan *Cross-Site Scripting (XSS)*. Setiap *endpoint* API yang bersifat privat dijaga oleh blok *middleware* yang bertugas membedah (`verify`) keabsahan struktur token tersebut sebelum memproses *request*.

## 3.3.2 Pemrosesan File (File Handling)

Pengelolaan berkas pengarsipan tidak menggunakan penyimpanan blob (seperti AWS S3), melainkan menyentuh akses *filesystem* secara langsung (direktori fisik server) dengan mekanisme berikut:
1. **Penerimaan Berkas (Multer)**
   Proses *upload* dokumen atau gambar dari antarmuka pengguna ditangkap oleh *middleware* `multer`. *Multer* dikonfigurasi untuk menyimpan *stream* biner tersebut ke dalam partisi khusus, sembari menjaga nama asli (*original name*) serta melacak struktur *parent path* dari hierarki folder tujuan.
2. **Kompilasi Direktori (Archiver)**
   Aplikasi mendukung pengunduhan massal dalam format folder. Ketika Client mengunduh satu folder penuh yang berisi banyak berkas atau sub-folder, backend akan mengeksekusi library `archiver`. Modul ini mengiterasi lintasan *path* rekursif dan menyatukannya (*zipping*) menjadi satu arsip berekstensi `.zip` yang kemudian dikirimkan kembali ke klien dalam bentuk *stream* memori (tanpa harus membuang kapasitas disk temporer server).

## 3.3.3 API dan Konektivitas (CORS)

Pertukaran *Application Programming Interface* (API) dilakukan menggunakan format struktur data JSON. Modul `cors` diimplementasikan dengan spesifikasi *origin* yang membatasi bahwa rute (API) ini hanya mengizinkan *request* yang datang dari *domain* Frontend yang bersangkutan, sehingga melindungi *endpoint* dari injeksi layanan pihak ketiga yang tidak berizin.
