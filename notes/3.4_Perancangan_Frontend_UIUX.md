# 3.4 Perancangan Antarmuka (Frontend UI/UX)

Sisi klien (frontend) dari sistem DAFA dikembangkan sebagai *Single Page Application* (SPA) menggunakan arsitektur antarmuka **React.js**. *Bundler* modern **Vite** digunakan dalam proses kompilasi kode komponen demi memangkas *load time* serta optimalisasi *Hot Module Replacement* selama masa pengembangannya.

## 3.4.1 Pendekatan Desain User Interface (UI)

Penulisan gaya (*styling*) pada komponen UI sepenuhnya dikontrol melalui modul utilitas **Tailwind CSS**. Pendekatan *utility-first* ini menghindari penumpukan klasifikasi CSS kustom (*bloatware*).
- **Desain Modern & Terstruktur**: Memanfaatkan prinsip *flexbox* dan *grid layout* untuk menghadirkan susunan (*layout*) tabel manajemen arsip dan hierarki *folder* yang rapi.
- **Hierarki Navigasi Logis**: Karena arsip disusun bertingkat (berakar/bersarang layaknya direktori komputer), maka dibuat komponen *Breadcrumb Navigation* sehingga Client mengetahui letak persis posisinya di dalam direktori virtual dan dapat memundurkan layar (*back track*) dengan satu sentuhan kursor.
- **Ikonografi**: Library *React Icons* disematkan untuk memberi penanda (*visual cues*) perbedaan mana ekstensi berupa dokumen (.pdf, .doc), foto, atau kumpulan *folder*. 

## 3.4.2 Logika User Experience (UX)

Sistem dirancang untuk memastikan Client tidak mengalami kebingungan atau disorientasi (User Experience) selama mengoperasikan aplikasi melalui implementasi:
1. **Manajemen Status Reaktif (State Management)**: Pergerakan antar lintasan folder terjadi secara reaktif tanpa perlu pemuatan ulang halaman penuh (*full page reload*). Status (*state*) tabel diikat dengan fungsi kaitan *React Hooks* (`useState` & `useEffect`) yang me-*request* kembali daftar *child file* segera sesudah terjadi pergantian direktori sasaran.
2. **Isolasi Tampilan Berbasis Jabatan**: Ini adalah UX fundamental sistem. Komponen frontend tidak akan merender dan tidak akan menawarkan tautan opsi ke sub-menu atau folder arsip dari jabatan yang mana Client tidak memiliki privilese, sehingga tampilan tabel tidak terkesan rumit dan padat (hanya menampilkan apa yang menjadi urgensi pekerjaan mereka).
3. **Transisi Dinamis**: Modul `framer-motion` dilibatkan untuk melangsungkan animasi mikro (*micro-interactions*), seperti efek modal peramban muncul mulus, pergeseran *sidebar*, hingga transisi masuk/keluar dari daftar *file* yang diunggah.

## 3.4.3 Konsep Progressive Web Apps (PWA)

Agar memiliki visibilitas layaknya program sistem operasi asli (Native Apps), plugin `vite-plugin-pwa` memfasilitasi aplikasi agar mampu dipasang secara lokal (*installed*) pada layar beranda perangkat *mobile* maupun *desktop*. Ini didukung dengan file *Manifest* yang mengonfigurasi ikon peluncuran.
