# 3.5 Skenario dan Cara Kerja Aplikasi

Bagian ini menguraikan tahapan sistematis terkait bagaimana aktor atau pengguna akhir mengoperasikan perangkat lunak aplikasi pengarsipan.

## 3.5.1 Skenario Pengaturan (Set-up) oleh Admin

Sebelum sistem dapat dimanfaatkan untuk manajemen arsip harian, **Admin** harus mengonfigurasinya terlebih dahulu melalui serangkaian tindakan berikut:
1. Admin mengakses portal utama (UI Frontend) dan mengisikan kolom kredensial (username dan password *superuser*).
2. Setelah otorisasi `jsonwebtoken` tervalidasi, halaman memuat Dasbor utama yang menampilkan ringkasan data.
3. Admin memanajemen tahun dengan masuk ke menu **Tahun Akademik**. Admin menambahkan periode baru (misal: "2025/2026") lalu menyetel parameter statusnya ke posisi 'Aktif'. Kueri `global_settings` di basis data akan berubah, menargetkan seluruh muatan halaman pada periode spesifik ini.
4. Admin beranjak ke menu **Manajemen Pengguna**, menambahkan nama dan membuat akun untuk para tenaga kepegawaian (Client).
5. Pada panel modifikasi akun, Admin wajib mendefinisikan *Role Assignment* (hak akses jabatan) kepada akun yang baru dibentuk (contoh: mencentang status 'Waka Kurikulum' untuk Bapak Budi). Pada database, rekaman ini dimasukkan pada relasi struktur tabel `client_access`.

## 3.5.2 Skenario Pengarsipan dan Retrievi Berkas oleh Client

Setelah pengaturan awal diselesaikan oleh admin, aplikasi difokuskan pada pengumpulan dan pengunduhan bukti fisik administratif oleh entitas Client:
1. Client (Misal: Staf Kurikulum) menuju laman portal *Login* aplikasi.
2. Setelah sukses menembus rute keamanan melalui otorisasi *cookie-based*, sistem mengalihkan arah (*redirect*) pada beranda antarmuka eksplorasi.
3. Komponen *React.js* mengirimkan *request* pemuatan folder berbekal ID token yang dimiliki Client.
4. Karena identitas terekam sebagai bagian dari Waka Kurikulum, *middleware* backend menolak pengiriman struktur direktori divisi Humas atau Sapras, dan sebaliknya hanya me-*render* pohon sub-folder (misal: "Modul Ajar", "Perangkat Kurikulum") milik Waka Kurikulum pada struktur tabel GUI.
5. Client dapat mengklik ikon folder "Modul Ajar" (yang mana memicu *trigger update state path* di sisi *frontend*).
6. Untuk mengambil arsip, Client memilih berkas berekstensi `.pdf` yang ada pada kolom, lalu menginisiasi aksi "Unduh". Backend memproses *stream file* tersebut, menyajikannya ke *browser* untuk di-terima pada penyimpanan komputer lokal klien.
7. Aplikasi ini melarang Client untuk menghapus atau menulis ulang tatanan folder yang bersifat *fixed* untuk menghindari inkonsistensi pelaporan tahun ajaran, proses manajemen arsip hierarki mendalam (*deep archiving*) tetap disentralisasikan di ruang kontrol Administrator.
